# Guide instruction

- 반드시 테스트 코드를 작성할 것. 특히 domain 은 **반드시** 작성한다.
- 코드 생성시 가독성에 중점을 둘 것
- 코드와 메서드는 이해하기 쉽도록 주석을 추가할 것.
- 리팩토링 전 sequential thinking으로 먼저 단계별 계획을 세울 것
    - TDD 방법론을 기초로 두고 생각 할 것.
    - 순차적 사고 도구를 써서 필요한 단계 수를 잡고, 각 단계별 리스크까지 포함해서 정리할 것.
        - 계획은 최소 6단계 이상으로 세울 것.
            - 각 계획의 판단 근거를 자세하게 작성할 것.
        - plan 은 markdown 파일로 만들어서 저장한다.
            - 생성된 파일은 `plan` 디렉토리 아래에 저장한다.
        - 코드 생성 시 추천되는 디자인 패턴을 3개를 찾는다. 그 후 가독성, 성능, 유지보수성 각 10점 만점 기준으로 평가해서 가
          장 점수가 높은 것을 적용한다.
            - 각 점수를 평가한 기준을 plan 에 기록한다.
    - 반드시 테스트에 대한 플랜을 작성한다.
- 생성 후 테스트 코드를 실행해서 문제가 없는지 검증한다.
